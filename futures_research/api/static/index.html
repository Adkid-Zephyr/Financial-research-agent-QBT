测试流程

推荐验收顺序

低代码前端
  1. 先点“刷新健康状态”，确认 API 已启动，若存储开启会显示 PostgreSQL 已接入。
  2. 点击“连接运行事件”或“连接批量事件”，让 WebSocket 保持在线。
  3. 在“单品种触发”中提交 `CF`、`AU` 或 `LC`，观察事件流是否出现 `run_started` 到 `run_completed`。
  4. 在“批量触发”中提交 `CF,M,AU,AG,CU,AL,LC,SI` 或勾选“全部品种”，观察批量事件。
  5. 点击“刷新报告列表”，确认数据库已落库；未配置数据库时页面会提示只支持流程验证。
  6. 需要正式部署时，再对照页面底部“使用手册”执行 `docker compose` 和 cron 验收。

使用手册

              
本地与部署后怎么用

            

          

          

            

              
本地开发

              
1. 运行 `uvicorn futures_research.api.app:app --reload`。

              
2. 打开当前页面，先检查健康状态，再连接 WebSocket。

              
3. 通过表单触发 `POST /runs` 或 `POST /batches`，观察事件流。

              
4. 若启用了数据库，再查询 `/reports` 验证落库。

            

            

              
Docker Compose

              
1. `cp .env.example .env`。

              
2. `docker compose up --build -d`。

              
3. 浏览器访问 `http://127.0.0.1:8080/`。

              
4. 若要做计划任务，执行 `chmod +x deploy/cron/run_batch.sh` 后交给宿主机 cron。

            

            

              
推荐验收点

              
1. `/healthz` 为 `ok`。

              
2. 单品种触发后出现 `run_completed`。

              
3. 批量触发后出现 `batch_completed`。

              
4. 若存储开启，报告列表能显示新生成的 `run_id` 和分数。
